Tengo un problema de persistencia en `HighlightViewer.jsx`. Cuando le doy reacción a un video, deslizo al siguiente y vuelvo al anterior, la reacción desaparece. Esto ocurre porque el componente reinicia su estado basándose en la prop `items` original, ignorando mis cambios recientes.

Por favor, soluciona esto implementando un **"Local Cache de Reacciones"**. Sigue estos pasos:

1. **NUEVO ESTADO DE CACHÉ:**
   - Crea un estado para guardar los cambios de la sesión:
     `const [reactionCache, setReactionCache] = useState({});`
   - Este objeto almacenará las reacciones por ID: `{ [itemId]: { userReaction: '...', reactions: {...} } }`.

2. **MODIFICAR EL `useEffect` DE CARGA:**
   - En el `useEffect` que se ejecuta cuando cambia el `item` o `activeIndex`, cambia la lógica de asignación inicial.
   - **Lógica:**
     - Primero busca si existe datos para este `item.id` en `reactionCache`.
     - Si existe en caché, usa ESOS datos para `setUserReaction` y `setReactionCounts`.
     - Si NO existe en caché, usa los datos originales de `item.userReaction` e `item.reactions`.

3. **MODIFICAR `handleReaction` (Escribir en Caché):**
   - Dentro de la función `handleReaction`, además de actualizar el estado visual inmediato y llamar a la API, **actualiza el `reactionCache`**.
   - Ejemplo conceptual:
     ```javascript
     setReactionCache(prev => ({
       ...prev,
       [item.id]: {
         userReaction: nuevoTipo,
         reactions: nuevosContadores
       }
     }));
     ```

4. **INTEGRIDAD DE DATOS:**
   - Asegúrate de que `item.id` exista. Si tus items usan URLs como ID, usa `item.url` o `item.id` como clave consistente.

**Objetivo:** Que al navegar entre videos (Swipe), el componente recuerde si ya reaccioné y muestre los contadores actualizados sin tener que volver a pedir datos al servidor.

El intento anterior con `useMemo` no solucionó el parpadeo del video. Al actualizar el estado de las reacciones, React sigue afectando al elemento de video.

Vamos a aplicar la estrategia de **Extracción de Componente Memoizado**. Por favor, refactoriza `frontend/src/features/highlights/HighlightViewer.jsx` siguiendo estos pasos estrictos:

1. **CREA UN SUB-COMPONENTE INTERNO:**
   - Define un nuevo componente llamado `ReelPlayer` (puedes definirlo en el mismo archivo, fuera del componente principal, o en uno nuevo si prefieres, pero mantengámoslo simple).
   - Este componente debe recibir SOLO las props esenciales para el video: `src`, `poster`, `isActive` (booleano), `onNext`, `onPrev`, `onTimeUpdate`.
   - **CRÍTICO:** Envuelve este componente en `React.memo`.

   ```javascript
   const ReelPlayer = memo(({ src, poster, isActive, onNext, onPrev, onTimeUpdate }) => {
      // Mueve aquí toda la lógica de useRef del video, useEffect de play/pause, y el JSX del <video>
      // ...
   }, (prevProps, nextProps) => {
      // Custom comparison (opcional pero recomendado):
      // Solo renderizar si cambia la URL o el estado de isActive
      return prevProps.src === nextProps.src && prevProps.isActive === nextProps.isActive;
   });
